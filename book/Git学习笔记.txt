#### 操作基础
1. 创建本地git仓库
	git init

2. 添加文件到本地git仓库提交队列
	git add 文件名

3. 删除本地git仓库提交队列中的文件
	git reset 文件名

4. 同步本地git仓库提交队列中的文件到本地git仓库
	git commit -m "本次提交的备注信息"

5. 查看工作区(本地仓库)的状态
	git status

6. 检出云端仓库数据到本地仓库
	git clone 云端仓库地址

7. 同步本地git仓库到云端仓库
	git push [云端仓库地址]

8. 拉取(同步)云端git仓库到本地仓库,并合并冲突文件。可以使用git status查看
	git pull

9. 查看文件差异，当云端仓库与本地仓库原数据不同步时，需要做对比修改
	git diff [文件名]

10.查看提交日志
	git log

11.查看当前版本之前的提交日志
	git reflog

11.回到过去
	git reset --hard 提交日志中的commit值


#### 进阶
1. git协作冲突处理
	当多人协作提交代码到云端时，若同时修改了同个文件，则会出现上传失败。此时需要使用 pull 命令拉取云端数据同步到本地，人工对比冲突数据，可以使用 diff 命令查看，修改冲突文件，留取需要保留的数据，在重新提交。
